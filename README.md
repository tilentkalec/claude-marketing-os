# claude-marketing-os

**AI-native marketing operating system. Configure once, run for any brand.**

Built on [Claude Code](https://claude.com/claude-code). Open-sourced by [Tilen Tkalec](https://github.com/tilentkalec) — CEO of [eMatura](https://ematura.si), building an AI-native marketing agency.

---

## What you get

A complete marketing function in one repo:

- **A CMO orchestrator** that handles cross-channel reporting, planning, and analysis
- **6 channel subagents** — content strategy, content production, social media, Meta Ads, Google Ads, email
- **Reusable skills** — keyword research, blog writing, email campaigns, ad analysis, trend research
- **Slash commands** — `/cmo-report`, `/cmo-analyze`, `/cmo-plan`
- **Single source of brand truth** — `client-config.md`, generated once by the onboarding skill

You bring API keys for the channels you run. Everything else is configured by talking to the agents.

## Quick start

```bash
git clone https://github.com/tilentkalec/claude-marketing-os.git
cd claude-marketing-os
cp .env.example .env       # fill in only the keys for channels you run
```

Then open the repo in Claude Code and say:

> "Onboard a new client"

The onboarding skill walks you through brand, markets, channels, KPIs, and seasonality, then writes `client-config.md`.

After that:

```
/cmo-report                 # weekly cross-channel review
/cmo-analyze "why did our HU traffic drop last week?"
/cmo-plan "grow paid signups 30% in Q3"
```

## Architecture

```
User
  │
  ▼
CMO orchestrator (AGENTS.md)
  │
  ├── reads client-config.md
  ├── fans out to channel agents (.claude/agents/)
  └── synthesizes back into a unified report/plan
        │
        ▼
   Channel agents load skills (skills/) and run their playbooks
```

- **Agents** are channel-shaped (`meta-ads`, `email`, etc.) — read brand config, load the right skill.
- **Skills** are pure playbooks — reusable across brands, no client-specific data.
- **`client-config.md`** is the only place brand context lives.

This separation is the point: fork the repo, run onboarding, and the agents work without modification.

## What lives where

```
.
├── AGENTS.md                    CMO orchestrator
├── CLAUDE.md                    project context for Claude Code
├── client-config.md             brand config (gitignored, generated by onboarding)
├── .env / .env.example          credentials
├── .claude/
│   ├── agents/                  6 channel subagents
│   └── commands/                /cmo-* slash commands
├── skills/                      playbooks (blog writing, email, ads, SEO, etc.)
├── data/                        traffic/revenue JSON (gitignored)
├── outputs/                     reports, plans, generated content (gitignored)
├── notes/                       persistent client-specific notes (gitignored)
├── scripts/                     your data-pulling scripts
└── docs/                        design docs
```

## What you bring

- **API keys** for whichever channels you run — Klaviyo, Meta Ads, Google Ads, etc. The `.env.example` shows the convention: `<CHANNEL>_<KEY_TYPE>_<MARKET_CODE>`.
- **Data scripts** for traffic and revenue. The repo doesn't ship with these — drop your own GA4 puller, Stripe puller, Woo puller, etc., into `scripts/` and dump JSON to `data/`. The CMO reads from `data/` and passes context to channel agents.

See `docs/data-sources.md` for the JSON contract.

## Channels supported out of the box

| Channel | Skill | Agent |
|---|---|---|
| Email (Klaviyo) | `skills/email-klaviyo.md` | `email` |
| Google Ads | `skills/google-ads.md` | `google-ads` |
| Meta Ads (Facebook + Instagram) | `skills/meta-ads.md` | `meta-ads` |
| Short-form social | `skills/head-of-content.md` | `social-media` |
| Blog/content | `skills/blog-post-writer.md`, `skills/content-pipeline.md` | `content-producer` |
| SEO/GEO/AEO | `skills/keyword-researcher.md`, `skills/seo-geo-aeo-analyzer.md` | `content-strategist` |

Channels not active in your `client-config.md` are simply skipped — no need to remove the agent files.

## Why this exists

Most marketing teams below ~$5M ARR can't justify the headcount of a full agency or in-house marketing org. AI agents collapse that cost. This repo is the OS for running one operator + AI as a CMO function.

I built this running multi-channel marketing for a real ed-tech brand across 5 markets, then ported it into a brand-agnostic template. The client work stays private; this repo is the skeleton.
